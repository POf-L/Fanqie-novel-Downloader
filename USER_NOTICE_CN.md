# 用户须知：应用窗口打开方式变更

## 重要更新

我们改进了应用的启动方式，现在即使在没有安装 Microsoft Edge WebView2 的系统上也能正常使用！

## 您会注意到什么？

### 场景 1：正常情况（推荐）
- ✅ 应用会在独立窗口中打开
- ✅ 界面清爽，无浏览器地址栏
- ✅ 这是最佳体验

### 场景 2：自动降级（同样好用）
如果您看到以下提示信息：
```
PyWebView 浏览器引擎初始化失败
自动切换到系统浏览器...
```

**不用担心！这是正常的。**

- ✅ 应用会自动在您的默认浏览器中打开
- ✅ 所有功能完全相同
- ✅ 下载速度和质量不受影响

## 为什么会出现这种情况？

某些 Windows 系统（特别是精简版或企业版）可能缺少 Microsoft Edge WebView2 组件。
我们的应用会智能检测这种情况并自动切换到系统浏览器模式。

## 如何获得独立窗口体验？

如果您想使用独立窗口而不是浏览器，只需安装 Edge WebView2：

1. 访问下载页面：https://go.microsoft.com/fwlink/p/?LinkId=2124703
2. 下载并安装
3. 重启应用

**但这不是必需的！** 浏览器模式同样好用。

## 常见问题

### Q: 为什么我的应用在浏览器中打开？
**A:** 您的系统可能缺少 Edge WebView2 组件，应用自动使用了浏览器模式。

### Q: 浏览器模式和窗口模式有什么区别？
**A:** 
- 窗口模式：没有地址栏，看起来更像原生应用
- 浏览器模式：在浏览器标签页中打开，有地址栏
- **功能完全相同**，只是外观略有不同

### Q: 浏览器模式会影响下载速度吗？
**A:** 不会！下载逻辑完全独立，不受显示方式影响。

### Q: 我能强制使用浏览器模式吗？
**A:** 可以！卸载 pywebview：
```bash
pip uninstall pywebview
```

### Q: 安全吗？
**A:** 完全安全！
- 所有数据都在本地处理
- 应用只监听本地端口 (127.0.0.1)
- 没有任何外部连接风险

## 技术详情（可选阅读）

应用使用以下启动顺序：
1. 尝试使用 PyWebView 创建独立窗口
2. 如果失败，自动切换到系统默认浏览器
3. 两种方式的后端服务完全相同
